## 当前待验证假设

这些是假设，不是结论：

- 小型硬件团队愿意先购买一次性人工复核服务，而不是立即订阅 SaaS。
- KiCad 用户更容易接受本地 CLI 和 GitHub Action，因为不需要上传设计文件。
- 首批高价值规则集中在电源入口、USB/串口/CAN/RS485、BOM 字段、Gerber/drill/position 完整性和制造 profile。
- 最破坏信任的误报是“泛泛而谈但无文件、对象或规则证据”的 high/critical 风险。
